에이전트 앱은 AI 에이전트를 주요 사용자로 삼아 구축된 서비스입니다. 사람이 클릭하는 시각적 UI를 렌더링하는 대신, 에이전트가 프로그래밍 방식으로 발견하고 이해하고 호출할 수 있는 구조화된 엔드포인트를 제공합니다.

## 에이전트가 앱을 발견하는 방법

에이전트는 잘 알려진 URL 경로인 `/.well-known/SKILL.md`에 호스팅된 `SKILL.md` 파일을 통해 서비스를 찾습니다. 웹 개발 경험이 있다면 이 패턴이 익숙할 수 있습니다.

- Apple은 유니버설 링크를 위해 `/.well-known/apple-app-site-association`을 사용합니다
- SSL 인증기관은 도메인 소유권 검증을 위해 `/.well-known/acme-challenge/`를 사용합니다
- OAuth는 검색을 위해 `/.well-known/openid-configuration`을 사용합니다

`/.well-known/` 규칙은 예측 가능한 위치에 메타데이터를 호스팅하는 표준 방법입니다. 에이전트 앱의 경우, `SKILL.md` 파일이 이곳에 있어야 에이전트가 `yourdomain.com/.well-known/SKILL.md`를 확인하여 찾을 수 있습니다.

### SKILL.md에 들어가는 내용

`SKILL.md` 파일은 앱이 무엇을 하는지, 어떤 엔드포인트를 제공하는지, 어떻게 사용하는지를 설명합니다. 개념적인 템플릿은 다음과 같습니다.

```markdown Title "SKILL.md 템플릿"
# 앱 이름

## 설명
앱이 하는 일을 평이한 언어로 설명합니다.

## 엔드포인트

### POST /api/action-name
- **설명:** 이 엔드포인트가 하는 일
- **입력값:** 예상하는 파라미터
- **출력값:** 반환하는 것
- **결제:** x402 결제 필요 여부 및 비용

## 인증
에이전트가 인증하는 방법 (예: SIWA, API 키, x402).
```

에이전트가 `SKILL.md`를 읽으면 서비스가 무엇을 제공하는지, 어떻게 상호작용하는지 이해합니다. 사람의 개입이 필요하지 않습니다.

## 에이전트 사용자를 위한 설계

에이전트는 사람이 아닙니다. 탐색하거나 추측하거나 시각적 단서를 해석하지 않습니다. 에이전트 앱을 구축할 때는 다음 가이드라인을 따르세요.

- **사이드 이펙트를 명시하세요**: 엔드포인트가 자금을 이동하거나 상태를 수정하거나 외부 액션을 트리거한다면 `SKILL.md`에 명확히 기재하세요
- **구조화된 JSON을 반환하세요**: 에이전트는 HTML이 아닌 JSON을 파싱합니다. 모든 엔드포인트는 일관된 필드명으로 구조화된 JSON 응답을 반환해야 합니다
- **유료 접근에는 x402를 지원하세요**: 서비스에 비용이 든다면 [x402 프로토콜](/ai-agents-ko/core-concepts/payments-and-transactions)을 구현하여 에이전트가 사전 등록 없이 요청별로 결제할 수 있도록 하세요
- **명확한 오류 응답을 사용하세요**: 에이전트가 실패를 우아하게 처리할 수 있도록 적절한 HTTP 상태 코드와 함께 설명적인 오류 메시지를 반환하세요

## 앱을 검색 가능하게 만들기

`SKILL.md`를 `/.well-known/SKILL.md`에 호스팅하고 [ERC-8004 레지스트리](/ai-agents-ko/core-concepts/identity-verification-auth)에 앱을 등록하세요. 에이전트 신원에 사용되는 동일한 레지스트리가 앱 검색에도 사용됩니다. 등록이 완료되면 에이전트가 레지스트리를 조회하고 서비스를 찾고 `SKILL.md`를 읽어 어떻게 상호작용할지 이해할 수 있습니다.

<Warning>
에이전트 앱 프레임워크는 실험적입니다. 표준이 발전함에 따라 API와 규칙이 변경될 수 있습니다.
</Warning>

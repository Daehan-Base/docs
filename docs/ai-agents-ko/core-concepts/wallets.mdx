## 에이전트에 전용 지갑이 필요한 이유

AI 에이전트는 프롬프트를 통해 임의의 개인키를 생성하여 직접 지갑을 만들 수도 있습니다. 하지만 이 방법은 안전하지 않습니다. 그 이유는 다음과 같습니다.

- 개인키가 에이전트의 대화 컨텍스트에 노출되어 로그, 캐시, 미들웨어를 통해 유출될 수 있습니다
- 에이전트의 프롬프트 기록에 접근할 수 있는 모든 시스템이 키를 추출할 수 있습니다
- 키가 유출되면 지갑의 모든 자금이 영구적으로 손실됩니다

전용 지갑 서비스는 에이전트의 런타임과 분리된 안전한 인프라에서 키를 관리하여 이 문제를 해결합니다. 에이전트는 개인키를 직접 보지 않고도 트랜잭션을 요청할 수 있습니다.

    **적합한 경우:** 사전 빌드된 스킬로 빠른 지갑 연동이 필요한 OpenClaw 또는 BANKR 기반 에이전트.

    **작동 방식:** BANKR 지갑 스킬을 설치하면 에이전트가 잔액 조회, 토큰 전송, <Tooltip tip="스마트 컨트랙트를 통해 대출·거래 등의 금융 서비스를 제공하는 프로토콜">DeFi 프로토콜</Tooltip> 상호작용 기능을 갖추게 됩니다. 키는 BANKR이 저장하고 관리합니다.

  </Tab>

    **적합한 경우:** Coinbase 수준의 보안이 적용된 프로덕션 지갑 인프라가 필요한 Agent SDK 기반 에이전트.

    **작동 방식:** CDP API를 통해 지갑을 생성합니다. 에이전트가 API로 트랜잭션을 요청하면 CDP가 서명과 브로드캐스팅을 처리합니다. 키는 Coinbase의 보안 엔클레이브에서 관리되며 에이전트가 직접 접근하지 않습니다.

    [시작하기 →](https://docs.cdp.coinbase.com/agentic-wallet/quickstart)

## 지갑으로 할 수 있는 것

에이전트가 지갑을 갖추면 다음과 같은 작업이 가능합니다.

- **<Tooltip tip="미국 달러 등 안정적인 자산에 연동된 암호화폐(예: USDC). 1 USDC는 항상 1달러와 같도록 설계됩니다.">스테이블코인</Tooltip> 보유**: USDC 등 토큰을 수신하고 저장
- **결제 송수신**: 다른 지갑으로 자금 전송, API 이용료 지불, 서비스 대금 수령
- **메시지 서명**: 특정 메시지나 요청이 에이전트로부터 온 것임을 암호학적으로 증명 (신원 확인에 활용)
- **프로토콜 상호작용**: <Tooltip tip="온체인에 배포되어 조건이 충족되면 자동으로 실행되는 프로그램. 토큰 스왑부터 대출까지 다양한 기능을 제공합니다.">스마트 컨트랙트</Tooltip>를 호출하여 토큰 스왑, 유동성 공급, 기타 온체인 서비스 이용

## x402: 스테이블코인 요청별 결제

x402를 사용하면 에이전트가 구독이나 API 키 없이 <Tooltip tip="미국 달러 등 안정적인 자산에 연동된 암호화폐(예: USDC). 1 USDC는 항상 1달러와 같도록 설계됩니다.">스테이블코인</Tooltip>으로 API 요청 비용을 지불할 수 있습니다. 에이전트가 리소스를 요청하면 서버가 가격을 알려주고, 에이전트가 결제하면 서버가 데이터를 전달합니다.

이름은 HTTP 상태 코드 `402 Payment Required`에서 따왔습니다. 이 코드는 1990년대부터 HTTP 스펙에 예약되어 있었지만 널리 사용된 적이 없었습니다. x402가 드디어 이를 실용적으로 활용합니다.

### x402 작동 방식

<Steps>
  <Step title="에이전트가 요청을 보냅니다">
    에이전트가 API 엔드포인트에 일반적인 HTTP 요청을 보냅니다. 다른 API 호출과 동일합니다.
  </Step>

  <Step title="서버가 402 Payment Required로 응답합니다">
    데이터를 반환하는 대신, 서버가 `402` 상태 코드와 함께 결제 세부사항을 응답합니다. 비용이 얼마인지, 어떤 스테이블코인으로 지불해야 하는지, 어디로 전송해야 하는지를 포함합니다.
  </Step>

  <Step title="에이전트가 결제합니다">
    에이전트의 지갑이 서버의 안내에 따라 온체인 결제를 구성하고 전송합니다. 사람의 승인 없이 자동으로 처리됩니다.
  </Step>

  <Step title="서버가 데이터를 전달합니다">
    결제가 확인되면 서버가 요청한 데이터를 반환합니다. 전체 흐름은 수 초 안에 완료됩니다.
  </Step>
</Steps>

이 방식은 에이전트에게 강력합니다. 에이전트와 서비스 사이에 사전 관계가 없어도 되기 때문입니다. 자금이 있는 지갑을 가진 에이전트라면 어떤 x402 지원 API든 즉시 결제하고 사용할 수 있습니다.

[x402에 대해 더 알아보기 →](https://www.x402.org/)

## 스킬: 에이전트가 수행할 수 있는 액션

스킬은 에이전트가 호출할 수 있는 액션을 정의하는 타입화된 함수 정의입니다. 각 스킬은 이름, 예상 입력값, 반환 타입을 지정합니다. LLM 툴 사용 연동의 도구 또는 함수 스키마와 유사합니다. 에이전트는 런타임에 이 정의를 파싱하고 관련이 있을 때 호출합니다.

스킬은 토큰 잔액 확인, 스왑 실행, 결제 전송, <Tooltip tip="조건이 충족되면 자동으로 실행되도록 온체인에 배포된 프로그램">스마트 컨트랙트</Tooltip> 호출 등의 액션을 담당합니다.

### 스킬의 형태

스킬 정의에는 액션 이름, 기능 설명, 필요한 입력값, 예상 출력값이 포함됩니다. 간략한 예시는 다음과 같습니다.

```json Title "개념적 스킬 정의"
{
  "skill": "check-balance",
  "description": "지갑 주소의 토큰 잔액을 조회합니다",
  "inputs": {
    "wallet_address": "조회할 주소",
    "token": "토큰 심볼 (예: USDC, ETH)"
  },
  "output": "지정한 토큰의 현재 잔액"
}
```

에이전트는 이 정의를 읽고 어떤 입력값이 필요한지 파악한 뒤, 해당 작업과 관련이 있을 때 스킬을 호출합니다.

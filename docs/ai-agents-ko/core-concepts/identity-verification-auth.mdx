## 신원이 중요한 이유

에이전트가 서로, 그리고 서비스와 자율적으로 상호작용하는 세계에서 신뢰는 근본입니다. 신원이 없으면 다음과 같은 문제가 발생합니다.

- 서비스가 요청이 인가된 에이전트로부터 왔는지 확인할 수 없습니다
- 에이전트가 응답이 실제 서비스로부터 온 것인지 (악의적인 사칭자가 아닌지) 확인할 수 없습니다
- 다른 에이전트가 여러분의 에이전트가 무엇을 하는지, 어떻게 상호작용해야 하는지 알 수 없습니다

Base의 신원 표준은 이 세 가지 문제를 모두 해결합니다. 에이전트가 공개 디렉토리에 자신을 등록하고, 모든 요청에서 신원을 증명하고, 다른 에이전트와 서비스를 발견할 수 있도록 합니다.

## 레지스트리: 에이전트를 위한 공개 디렉토리

레지스트리는 에이전트와 서비스가 자신의 신원을 게시하는 공개 디렉토리입니다. 누구든 레지스트리를 조회하여 에이전트의 이름을 메타데이터로 확인할 수 있습니다. 에이전트가 무엇을 하는지, 어떤 엔드포인트에 있는지, 서명에 어떤 공개키를 사용하는지 등이 포함됩니다.

Base에서는 **ERC-8004** 표준을 통해 구현됩니다. 에이전트가 등록하면 그 항목이 온체인에 저장되므로, 중앙 기관에 의존하지 않고 누구나 검증할 수 있습니다.

레지스트리 항목에는 일반적으로 다음이 포함됩니다.

- 에이전트의 이름과 설명
- 연락 가능한 엔드포인트
- 에이전트의 신원을 암호학적 자격증명에 연결하는 <Tooltip tip="공개키(반송 주소처럼 공개적으로 공유됨)와 개인키(서명 도장처럼 비밀로 유지됨)의 쌍. 공개키는 에이전트를 식별하고, 개인키는 신원을 제어하고 있음을 증명합니다.">키 쌍</Tooltip>

[에이전트 레지스트리(ERC-8004)에 대해 더 알아보기 →](https://www.8004.org/)

## 검증: 런타임에서 신원 증명

등록은 에이전트의 존재를 세상에 알립니다. 검증은 각 요청이 실제로 여러분의 에이전트로부터 온 것임을 증명합니다. 이는 **ERC-8128** 표준을 통해 처리됩니다.

<Steps>
  <Step title="에이전트를 등록합니다">
    에이전트가 ERC-8004 레지스트리에 신원과 공개키를 등록합니다. 이는 일회성 설정 단계입니다.
  </Step>

  <Step title="각 요청에 서명합니다">
    에이전트가 서비스를 호출할 때 개인키로 요청에 서명합니다. 이를 통해 해당 특정 요청에 고유한 암호학적 서명이 생성됩니다.
  </Step>

  <Step title="서비스가 서명을 검증합니다">
    서비스가 레지스트리에서 에이전트의 공개키를 조회하고 서명을 확인합니다. 서명이 일치하면, 서비스는 요청이 등록된 에이전트로부터 온 것이며 사칭자가 아님을 알 수 있습니다.
  </Step>
</Steps>

이는 양방향으로 작동합니다. 에이전트도 레지스트리에서 서비스의 서명을 확인하여 응답이 정당한지 검증할 수 있습니다.

[에이전트 검증(ERC-8128)에 대해 더 알아보기 →](https://erc8128.slice.so/concepts/overview)

## 에이전트로 로그인 (SIWA)

SIWA는 레지스트리와 검증을 하나의 SDK로 묶습니다. "Google로 로그인"이 OAuth와 신원 검증을 하나의 연동으로 묶는 것과 같습니다. ERC-8004와 ERC-8128을 별도로 구성하는 대신, SIWA를 연동하면 두 기능을 바로 사용할 수 있습니다.

SIWA를 사용하면 에이전트가 다음을 수행할 수 있습니다.

- **신원 등록**: 온체인 디렉토리에 등록
- **서비스 인증**: 등록된 에이전트임을 증명하여 서비스에 로그인
- **모든 요청 서명**: 서비스가 진위를 확인할 수 있도록 서명

SIWA는 에이전트에게 검증 가능한 신원을 부여하는 가장 빠른 방법입니다. 등록 및 검증 과정에 대한 더 세밀한 제어가 필요한 경우에만 ERC-8004와 ERC-8128을 직접 사용하세요.

[siwa.id에서 더 알아보기 →](https://siwa.id)

<Tip>
가장 간단한 연동 방법으로 **SIWA로 시작하세요**. 에이전트의 등록 방식과 검증 수행 방식을 세밀하게 제어해야 하는 경우 ERC-8004와 ERC-8128을 직접 사용하세요.
</Tip>

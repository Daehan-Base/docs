에이전트가 다른 에이전트 및 서비스와 상호작용하려면, 신뢰할 수 있는 신원 체계가 필요합니다. Base는 에이전트가 자신의 정체를 증명하고, 상대방을 검증하며, 서비스에 인증하기 위한 도구를 제공합니다.

## 에이전트 신원이 중요한 이유

AI 에이전트의 수가 증가하면서, 다음과 같은 질문이 중요해지고 있습니다:

- 이 에이전트가 누구를 대신하여 행동하는가?
- 이 에이전트를 신뢰할 수 있는가?
- 이 에이전트에게 어떤 권한이 부여되었는가?

신원 체계가 없으면, 에이전트 간의 상호작용에서 사기, 무단 접근, 그리고 책임 추적 불가능 등의 문제가 발생합니다.

## 에이전트 디렉토리

Base는 에이전트를 공개 디렉토리에 등록할 수 있는 체계를 제공합니다. 이를 통해:

- **디스커버리**: 다른 에이전트와 서비스가 여러분의 에이전트를 찾을 수 있습니다
- **검증**: 에이전트의 정체와 능력을 온체인에서 확인할 수 있습니다
- **평판**: 에이전트의 활동 이력과 신뢰도를 추적할 수 있습니다

## 인증 방식

### 지갑 기반 인증

에이전트의 지갑 주소가 기본적인 신원 역할을 합니다. 에이전트가 트랜잭션에 서명하면, 이 서명이 신원 증명이 됩니다.

```typescript
// 에이전트 신원 증명 예시
const message = "이 에이전트는 서비스 X에 접근을 요청합니다";
const signature = await wallet.signMessage(message);

// 서비스 측에서 서명 검증
const isValid = verifySignature(message, signature, agentAddress);
```

### 위임 체인 (Delegation Chain)

사용자가 KYC를 완료하면, 그 인증 상태를 에이전트에게 위임할 수 있습니다. 이를 통해:

1. 사용자가 한 번 KYC를 완료합니다
2. 에이전트에게 위임 자격증명(delegated credential)을 발급합니다
3. 에이전트의 모든 트랜잭션이 검증된 사용자에게 추적 가능합니다

### Attestation

온체인 attestation을 통해 에이전트의 속성과 자격을 증명할 수 있습니다:

- 에이전트가 특정 조직에 속해있음을 증명
- 에이전트의 기능과 권한 범위를 정의
- 시간 제한이 있는 접근 권한을 부여

## 보안 고려사항

<Warning>
에이전트 신원 관리에서 주의해야 할 사항들:
</Warning>

- 에이전트의 권한을 최소한으로 설정하세요 (최소 권한 원칙)
- 위임된 자격증명에 시간 제한을 설정하세요
- 에이전트의 활동을 정기적으로 감사하세요
- 필요시 즉시 자격증명을 취소할 수 있는 메커니즘을 구축하세요

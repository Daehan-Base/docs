---

Base에서 제품이 잘 작동하려면 단순한 체인 배포를 넘어, 어떤 사용자 경험이 온체인에서 더 잘 작동하는지까지 제품 전략에 반영해야 합니다. 이 단계에서는 내러티브, 기술 스택, 유통 경로를 함께 정리합니다.

## 1. 포지셔닝과 one-liner

- 제품의 핵심 가치를 한 문장으로 정의합니다
- 왜 이 경험이 Base에서 더 강해지는지 설명할 수 있어야 합니다
- “온체인이라서 가능한 경험”이 무엇인지 구체화합니다

강한 one-liner는 제품 소개, 커뮤니티 포스트, 런치 문서 전반의 기준점이 됩니다.

## 2. 생태계 트렌드 관찰

- [Base Leaderboard](https://www.base.dev/leaderboard)를 보며 어떤 앱이 주목받는지 확인합니다
- 비슷한 카테고리의 제품이 어떤 메시지와 UX를 쓰는지 살펴봅니다
- 런치 시점에 맞는 경쟁 포지셔닝과 차별화 포인트를 정리합니다

## 3. Base 기술 스택 선택

제품 유형에 따라 아래 기본 레이어를 조합합니다.

- [Base Chain](/base-chain/quickstart/why-base): 빠르고 저렴한 기본 네트워크
- [Base Account](/base-account/overview/what-is-base-account): 스마트 월렛 기반 온보딩과 트랜잭션 UX
- [OnchainKit](/onchainkit/latest/getting-started/overview): 프론트엔드에서 온체인 UX를 빠르게 구현할 때 유용한 컴포넌트/유틸리티
- [Mini Apps](/mini-apps/quickstart/create-new-miniapp): Base App 안에서 배포 가능한 소비자 경험

기존 웹 또는 모바일 제품을 확장하는 팀이라면 [기존 Mini App 마이그레이션](/mini-apps/quickstart/migrate-existing-apps)도 검토할 가치가 있습니다.

## 4. AI 에이전트와 확장 기능

제품에 자동화, 보조 기능, 온체인 액션 실행 레이어가 필요하다면 [AI 에이전트 가이드](/ai-agents-ko/index)를 함께 참고하세요. 결제, 지갑, 신원, agent app 구조를 별도 레이어로 얹을 수 있습니다.

## 5. 추가 참고 리소스

- [Builder Services Hub](/get-started/base-services-hub): 인프라, 데이터, GTM 파트너 탐색
- [paymasterService](/base-account/reference/core/capabilities/paymasterService): 가스리스 경험 설계
- [Build an App](/get-started/build-app): 빠른 프로토타입 및 개발 시작점

## 6. 이 단계의 결과물

- 제품 소개 one-liner
- Base 맞춤형 UX/온체인 설계 초안
- 사용 예정 기술 스택 목록
- 런치 전 보여줄 데모 또는 핵심 사용자 흐름

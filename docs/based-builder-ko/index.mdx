이 가이드는 Base 위에서 프로덕트를 빌딩하려는 팀, 창업자, 개발자를 위한 한국어 온보딩 문서입니다. 목표는 특정 커뮤니티 프로그램에 의존하지 않더라도 Base 생태계에 자연스럽게 들어와 제품 전략, 온체인 정체성, 런치 준비를 체계적으로 정리할 수 있게 돕는 것입니다.

<Tip>
이 가이드는 [Grampus](https://x.com/gram_voyage)가 Base 한국 커뮤니티를 위해 작성했습니다.
</Tip>

## 이 가이드가 다루는 것

- **생태계 적합성 정리**: 지금 만들고 있는 제품이 Base에서 어떤 유저와 빌더에게 가치를 주는지 정리합니다
- **온체인 정체성 구축**: 프로젝트 계정, 파운더 프로필, 빌더 코드, 소셜 존재감을 갖춥니다
- **제품 전략 구체화**: Base 고유의 UX와 배포 환경을 반영해 포지셔닝, 기술 스택, 성장 루프를 설계합니다
- **런치 준비**: 문서, 데모, 런치 내러티브, 그리고 Base App에서 배포되는 Mini App인 경우 Featured 제출 준비까지 실제 배포 직전 체크리스트를 점검합니다

## 왜 Base에서 빌드하나요

- **빌더 친화적 스택**: [Base Chain](/base-chain/quickstart/why-base), [Base Account](/base-account/overview/what-is-base-account), [Mini Apps](/mini-apps/quickstart/create-new-miniapp) 등 제품 개발에 바로 연결되는 기본 레이어가 잘 갖춰져 있습니다
- **제품 중심 배포 경로**: 소비자 앱, 미니앱, 온체인 UX, 에이전트 경험까지 하나의 제품 전략 안에서 연결할 수 있습니다
- **성장 지원 자원**: [Base Mentorship Program](/get-started/base-mentorship-program), [Country Leads & Ambassadors](/get-started/country-leads-and-ambassadors), [Builder Services Hub](/get-started/base-services-hub) 같은 지원 채널을 활용할 수 있습니다
- **글로벌 가시성**: 빌드 인 퍼블릭, 빌더 코드 기반 기여도 귀속, 그리고 Base App에서 배포되는 Mini App인 경우 Featured 제출 등 생태계 안에서 제품을 드러낼 수 있는 경로가 있습니다

## 4단계 로드맵

1. **STEP 1 — Alignment**
프로덕트의 방향성과 Base 생태계 적합성을 빠르게 점검합니다.

2. **STEP 2 — Onchain Identity**
프로젝트와 파운더의 온체인 정체성을 갖추고, 생태계에서 발견 가능한 상태를 만듭니다.

3. **STEP 3 — Product Strategy**
내러티브, UX, 기술 스택, 유통 경로를 Base에 맞게 구체화합니다.

4. **STEP 4 — Launch**
런치 전 문서, 데모, 프로모션, 그리고 Base App에서 배포되는 Mini App인 경우 Featured 제출까지 공개 출시를 준비합니다.

Agent App은 AI 에이전트를 주요 사용자로 설계된 서비스입니다. 사람을 위한 UI 대신, 에이전트가 쉽게 소비할 수 있는 구조화된 API 엔드포인트를 제공합니다.

## Agent App이란?

전통적인 웹 애플리케이션이 사람을 위해 UI를 제공하는 것처럼, Agent App은 AI 에이전트를 위해 설계됩니다:

- **구조화된 엔드포인트**: 에이전트가 이해하고 호출할 수 있는 명확한 API를 노출합니다
- **머신 리더블 응답**: JSON 등 구조화된 형식으로 데이터를 반환합니다
- **자동 결제**: x402를 통해 서비스 사용에 대한 자동 과금을 지원합니다
- **디스커버리**: 다른 에이전트가 서비스를 찾고 사용할 수 있도록 등록합니다

## Agent App을 구축해야 하는 이유

AI 에이전트 생태계가 성장하면서, 에이전트들은 다양한 서비스에 접근해야 합니다:

- **데이터 제공자**: 시장 데이터, 뉴스 피드, 분석 리포트 등을 에이전트에게 제공
- **액션 실행기**: 에이전트의 요청에 따라 특정 작업을 수행 (예: 주문 실행, 예약 처리)
- **인프라 서비스**: 컴퓨팅, 스토리지, AI 추론 등의 리소스를 에이전트에게 제공

## 구축 방법

### 1. API 엔드포인트 설계

에이전트가 쉽게 이해하고 사용할 수 있는 엔드포인트를 설계하세요:

```typescript
// Agent App API 예시
app.get("/api/market-data/:token", async (req, res) => {
  const { token } = req.params;
  const data = await getMarketData(token);
  
  res.json({
    token,
    price: data.price,
    volume24h: data.volume,
    change24h: data.change,
    timestamp: Date.now(),
  });
});
```

### 2. x402 결제 통합

서비스에 대한 자동 결제를 설정하세요:

```typescript
// x402 미들웨어 적용
import { x402Middleware } from "@coinbase/x402";

app.use("/api/premium", x402Middleware({
  amount: "0.01",     // 요청당 0.01 USDC
  currency: "USDC",
  recipient: "0x...", // 수금 주소
}));
```

### 3. 디스커버리 등록

다른 에이전트가 여러분의 서비스를 찾을 수 있도록 디렉토리에 등록하세요:

- 서비스 이름과 설명
- 제공하는 기능 목록
- API 엔드포인트와 사용 방법
- 가격 정보

### 4. 에이전트 친화적 문서 제공

에이전트가 서비스를 자동으로 이해하고 사용할 수 있도록:

- OpenAPI/Swagger 스펙을 제공하세요
- `llms.txt` 파일로 AI 친화적인 문서를 작성하세요
- 명확한 에러 메시지와 응답 코드를 사용하세요

## 실제 활용 사례

| 유형 | 예시 | 설명 |
| --- | --- | --- |
| 데이터 서비스 | NFT 바닥가 API | 에이전트가 NFT 컬렉션의 실시간 바닥가를 조회 |
| 트레이딩 서비스 | DEX 집계기 | 에이전트가 최적의 스왑 경로를 찾아 실행 |
| 분석 서비스 | 온체인 분석 | 에이전트가 지갑/프로토콜 활동을 분석 |
| 인프라 | AI 추론 API | 에이전트가 AI 모델에 추론 요청을 전송 |

## 모범 사례

<Tip>
Agent App을 구축할 때 유의사항:
</Tip>

- **명확한 API 설계**: 에이전트가 쉽게 파싱할 수 있는 일관된 응답 형식을 사용하세요
- **적절한 에러 처리**: 에이전트가 에러를 이해하고 대응할 수 있도록 구조화된 에러 메시지를 반환하세요
- **속도 제한(Rate Limiting)**: 남용을 방지하기 위해 요청 제한을 설정하세요
- **버전 관리**: API 변경 시 하위 호환성을 유지하세요
- **모니터링**: 에이전트의 사용 패턴과 성능을 추적하세요

AI 에이전트가 온체인에서 활동하려면 자체적인 지갑이 필요합니다. 지갑은 에이전트가 스테이블코인과 토큰을 보유하고, 결제를 전송하며, 트랜잭션에 서명할 수 있는 핵심 인프라입니다.

## 에이전트에게 전용 지갑이 필요한 이유

에이전트는 사람의 지갑을 직접 사용하는 것이 아니라 자체적인 전용 지갑을 가져야 합니다. 이유는 다음과 같습니다:

- **자율성**: 에이전트가 사람의 승인 없이도 트랜잭션을 실행할 수 있습니다
- **보안 격리**: 에이전트의 자금이 사용자의 개인 자금과 분리됩니다
- **권한 제어**: 에이전트의 지출 한도와 허용된 작업을 프로그래밍 방식으로 설정할 수 있습니다
- **추적성**: 모든 에이전트 트랜잭션이 별도로 기록되어 감사가 용이합니다

## 지갑 옵션

### CDP Wallet

Coinbase Developer Platform(CDP)에서 제공하는 지갑 인프라입니다. Trusted Execution Environment(TEE)에서 보안을 유지하는 비수탁형(non-custodial) 지갑을 제공합니다.

**주요 특징:**
- 엔터프라이즈급 보안
- 내장 KYT(Know Your Transaction) 스크리닝
- x402 프로토콜과의 네이티브 통합
- 가스리스(gasless) 트랜잭션 지원

```typescript
// CDP Wallet 생성 예시
import { AgentKit } from "@coinbase/agentkit";

const agentKit = await AgentKit.from({
  cdpApiKeyName: process.env.CDP_API_KEY_NAME,
  cdpApiKeyPrivateKey: process.env.CDP_API_KEY_PRIVATE_KEY,
});
```

### 서드파티 지갑 Provider

Agent SDK는 지갑에 구애받지 않으므로(wallet-agnostic), 다음과 같은 다른 provider도 사용할 수 있습니다:

- **Privy** — 소셜 로그인과 임베디드 지갑에 특화
- **Viem** — 저수준 블록체인 상호작용을 위한 경량 라이브러리

### Smart Wallet (계정 추상화)

Smart Contract Wallet은 에이전트에게 추가적인 유연성을 제공합니다:

- **다중 키 아키텍처**: 소유자 키와 에이전트 키를 분리하여 보안을 강화합니다
- **프로그래밍 가능한 가드레일**: Smart Contract 수준에서 지출 한도와 허용된 작업을 정의합니다
- **배치 트랜잭션**: 여러 작업을 하나의 트랜잭션으로 묶어 실행합니다

## 보안 모범 사례

<Warning>
프로덕션 환경에서는 반드시 지갑 보안에 주의하세요.
</Warning>

- 프라이빗 키를 코드에 하드코딩하지 마세요
- 환경 변수나 시크릿 관리 서비스를 사용하세요
- 에이전트의 지출 한도를 적절히 설정하세요
- 정기적으로 에이전트 트랜잭션을 모니터링하세요
- 테스트넷(Base Sepolia)에서 충분히 테스트한 후 메인넷으로 이동하세요

Base에서 AI 에이전트를 구축할 때 세 가지 주요 프레임워크 중에서 선택할 수 있습니다. 각 프레임워크는 서로 다른 수준의 제어, 커스터마이징, 호스팅 옵션을 제공합니다.

## Agent SDK

Agent SDK는 가장 많은 제어권을 제공하는 자체 호스팅 옵션입니다. Coinbase Developer Platform(CDP)의 AgentKit을 기반으로 하며, 에이전트를 직접 구축하고 자체 인프라에서 실행합니다.

**적합한 경우:**
- 에이전트 로직에 대한 완전한 제어가 필요한 경우
- 자체 서버 또는 클라우드 환경에서 호스팅하고 싶은 경우
- LangChain, Vercel AI SDK 등 기존 AI 프레임워크와 통합하고 싶은 경우

<Info>
Agent SDK는 프레임워크에 구애받지 않도록(framework-agnostic) 설계되어 있어, 어떤 AI 프레임워크와도 함께 사용할 수 있습니다. 또한 지갑에 구애받지 않아(wallet-agnostic) CDP, Privy, Viem 등 다양한 지갑 provider를 선택할 수 있습니다.
</Info>

```bash
# 빠른 시작
npm create onchain-agent@latest
```

## OpenClaw

OpenClaw는 오픈소스 AI 어시스턴트로, 사전 구성된 온체인 기능이 포함되어 있습니다. 에이전트를 빠르게 시작하되, 소스 코드에 대한 접근은 유지하고 싶은 경우 적합합니다.

**적합한 경우:**
- 빠른 프로토타이핑과 실험
- 온체인 기능이 내장된 즉시 사용 가능한 에이전트
- 오픈소스 커뮤니티 기반의 도구

## BANKR

BANKR는 관리형 API 서비스로, 인프라 관리 없이 에이전트를 운영할 수 있습니다.

**적합한 경우:**
- 인프라 설정 없이 바로 시작하고 싶은 경우
- 에이전트 호스팅과 운영을 위임하고 싶은 경우
- 최소한의 설정으로 빠르게 배포하고 싶은 경우

## 프레임워크 비교

| 기능 | Agent SDK | OpenClaw | BANKR |
| --- | --- | --- | --- |
| 호스팅 | 자체 호스팅 | 자체 호스팅 | 관리형 |
| 커스터마이징 | 높음 | 중간 | 낮음 |
| 설정 난이도 | 중간 | 낮음 | 최소 |
| 소스 코드 접근 | 전체 | 전체 | 제한적 |
| 적합한 대상 | 프로덕션 앱 | 프로토타이핑 | 빠른 배포 |
